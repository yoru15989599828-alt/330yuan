🚀 Railway 部署教程（最推荐！）

🚀 方案一：使用 Railway（最推荐！） 不需要绑卡

为什么选 Railway？

  • ✅ 完全不需要信用卡！
  • ✅ 功能强大，自动检测项目类型

第一步：准备工作

1

📧 注册 GitHub 账号（如果已有，跳过）

Railway 通过 GitHub 登录，需要先准备 GitHub 账号。

  1. 访问：https://github.com
  2. 点击 Sign up 注册
  3. 验证邮箱
2

📥 准备服务器代码（如果已有，跳过）

把服务器代码上传到 GitHub，确保代码仓库包含：

  • server.js - 服务器主文件
  • package.json - 依赖配置文件

💡 package.json 示例

{ "name": "my-chat-server", "version": "1.0.0", "main": "server.js", "scripts": { "start": "node server.js" }, "dependencies": { "ws": "^8.0.0" } }

第二步：在 Railway 部署

3

🌐 注册 Railway 账号

  1. 访问：https://railway.app
  2. 点击 Login（登录）
  3. 选择 Login with GitHub（用 GitHub 登录）
  4. 授权 Railway 访问你的 GitHub
✅ 不需要填写任何支付信息！直接用 GitHub 登录即可！
4

🚀 创建新项目并部署

  1. 登录后，点击 New Project（新建项目）
  2. 选择 Deploy from GitHub repo（从 GitHub 仓库部署）
  3. 第一次使用需要点击 Configure GitHub App 授权
  4. 在列表中找到你的代码仓库，点击它
  5. 点击 Deploy Now（立即部署）
🎉 Railway 会自动检测到这是 Node.js 项目，自动安装依赖并启动！
5

⚙️ 配置环境变量（可选）

如果你的服务器代码需要设置端口，Railway 会自动提供 PORT 环境变量。

确保你的 server.js 中有类似这样的代码：

const PORT = process.env.PORT || 8080; server.listen(PORT, () => { console.log(`Server is running on port ${PORT}`); });
Railway 会自动分配端口，你的代码需要读取 process.env.PORT
6

🌍 生成公开域名

部署成功后，需要生成一个公开域名：

  1. 在项目页面，点击你的服务（通常显示为仓库名）
  2. 进入服务详情页面
  3. 点击 Settings（设置）标签
  4. 找到 Networking 或 Domains 部分
  5. 点击 Generate Domain（生成域名）
  6. Railway 会自动分配一个域名，比如：
    my-chat-server.up.railway.app

第三步：获取服务器地址

7

📋 组装 WebSocket 地址

把生成的域名改成 WebSocket 格式：

  1. 如果域名是：my-chat-server.up.railway.app
  2. WebSocket 地址就是：
    wss://my-chat-server.up.railway.app
    （在前面加 wss://）
  3. 如果代码有路径（比如 /ws），要加上：
    wss://my-chat-server.up.railway.app/ws
  4. 复制这个地址

第四步：测试连接

8

🧪 在聊天程序中测试

  1. 打开聊天程序
  2. 进入"设置" → "联机功能"
  3. 打开"启用真人联机"开关
  4. 在"服务器地址"输入框粘贴刚才的地址
  5. 点击"连接服务器"
  6. 显示 "已连接" 就成功了！

Q3A: 我不想绑卡，用哪个平台？
A: 强烈推荐按以下顺序选择：
  1. Railway - 完全不需要绑卡，功能强大，自动配置（最推荐！）
  2. Zeabur - 完全不需要绑卡，性能好，中文支持
  3. Glitch - 完全不需要绑卡，老牌稳定
  4. Replit - 完全不需要绑卡，在线编辑器
避免使用 Render，它现在可能需要绑定信用卡。
Q4: 如何让免费服务器不休眠？
A:
  • Railway：本身不会频繁休眠，无需额外设置
  • Zeabur：本身不会频繁休眠，无需额外设置
  • Glitch/Replit/Render：可以使用 UptimeRobot（https://uptimerobot.com）免费服务，每5分钟自动访问一次你的服务器 https:// 地址（不是 WebSocket 地址），保持活跃状态

设置 UptimeRobot 的步骤：

  1. 注册 UptimeRobot 账号（免费）
  2. 添加新监控，类型选 HTTP(s)
  3. URL 填你的服务器 https:// 地址（不是 wss://）
  4. 监控间隔设置为 5分钟
  5. 保存即可
Q5: 我部署好了，朋友连接不上？
A: 检查这几点：
  • 地址是否正确（wss:// 或 ws:// 开头）
  • 端口号是否正确
  • 服务器是否正在运行（在 Render 看是否显示 ● Live）
  • 如果用的是云服务器，检查防火墙是否开放了端口
  • 如果用的是家里电脑，检查路由器端口转发是否设置正确
Q6: 可以换服务器吗？
A: 可以！但是换服务器后需要重新添加好友，因为好友列表是和服务器绑定的。建议先和朋友们商量好再换。
Q7: 服务器会保存聊天记录吗？
A: 不会！聊天记录只保存在你的手机上。服务器只负责实时传递消息，不存储任何聊天内容。这就是为什么换手机后聊天记录会丢失。
Q8: 可以多个人用同一个服务器吗？
A: 完全可以！这就是服务器的作用。只要大家都填同一个服务器地址，就可以互相添加好友、一起聊天。
Q9: 我不小心把服务器关了怎么办？
A:
  • Render：在控制台点击 Manual Deploy 重新部署
  • 云服务器：重新连接服务器，运行启动命令
  • 家里电脑：重新打开命令行，运行启动命令
Q10: 服务器费用太贵了，有更便宜的吗？
A: 方案一（Render）完全免费！如果非要买服务器，国内的轻量应用服务器最便宜，10元/月起。不要买配置太高的，1核1GB就够用了。
